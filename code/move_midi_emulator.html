Ableton Move MIDI Emulator

MIDI Output (Controls to Host)

All control messages are sent on ch16 (MIDI channel 16).

Pads (4x8 grid) send Note On/Off with velocity mapped to vertical press position. Aftertouch is sent as Key Pressure while held. Note numbers: 68-99 (bottom-left pad = nn68, top-right = nn99).

Step buttons (16) send Note On/Off nn16-31, velocity 127.

Encoders (8) send Note On/Off nn0-7 for touch, and relative CC 71-78 for turning. Relative values use 2's complement 7-bit: CW = 1-63, CCW = 65-127.

Volume knob sends Note On/Off nn8 (touch) and relative CC 79 (turn).

Scroll wheel sends Note On/Off nn9 (touch), relative CC 14 (turn), and CC 3 (press).

Buttons (function, nav, track, play, record, back, session) send CC with value 127 on press and 0 on release.

MIDI/SysEx Input (Host to LEDs)

The Move has two types of LEDs controlled by different protocols:

RGB LEDs are set via SysEx messages with 14-bit color values per channel:

F0 00 21 1D 01 01 3B [ch<<4] [index] [rLo] [rHi] [gLo] [gHi] [bLo] [bHi] F7

Each color component is 14-bit: value = (lo & 0x7F) | ((hi & 0x7F) << 7). RGB LEDs can also be set via Note On (nn) or CC using an indexed color table (value 0-127) [see below].

B/W LEDs are set via CC on ch16 using the color table's white-balance value for brightness.

Overlay: midi

Shows the MIDI note number (nn) or CC number (cc) for each control. Color coding: cyan = note (touch/press), yellow = CC (turn), red = encoder touch, purple = button CC.

Overlay: sysex

Shows LED addressing for each control. The icon marks RGB LEDs; marks B/W LEDs.

RGB labels show two values in orange:

  • D{n}:I{n} — LED driver destination and index (used by the RNBO leddriver mapdest() function)
  • ch{n}:{n} — SysEx channel and LED index in the SysEx message

Indexed Color Table (0-127)

MIDI value 0 = LED off. Values 1-127 map to an RGB color (for color LEDs) and a white-balance brightness (WB, for B/W LEDs). Values 65-116 are dim variants cycling through the spectrum in pairs (brighter/darker).
